## §0. DNA ของความ "ว้าว" (สกัดจาก reference — ทุกเทมเพลตต้องมี)

1. **รูปถ่ายจริงคือ 60–70% ของความสวย** — ทุกจุดที่มีรูป ต้องเป็นภาพถ่ายคน/สินค้าจริง โทนเดียวกันทั้งเว็บ (§1) ห้ามมี placeholder สี/กล่องเทา/ภาพ generate ที่ดูไม่ใช่ภาพถ่าย แม้แต่จุดเดียว — ถ้ารูปไม่พร้อม ให้หยุดงานส่วน UI แล้วไปจัดรูปให้เสร็จก่อน
2. **พื้นขาวจริง (#FFFFFF)** ตัวหนังสือเข้มเกือบดำ ความพรีเมียมมาจากความสะอาด+ระเบียบ ไม่ใช่พื้นครีม/สีพิเศษ
3. **ความหนาแน่นของ trust elements** — แถบ USP มีไอคอน (ส่งฟรี/คืนได้/จ่ายปลอดภัย/บริการลูกค้า), ดาวรีวิว, ยอดรีวิวในวงเล็บ, badge (ใหม่/-20%/BEST), โลโก้ช่องทางจ่ายเงิน — สิ่งเหล่านี้แหละที่ทำให้คนไทยรู้สึก "เว็บจริง น่าเชื่อถือ"
4. **Product card อัดข้อมูลแบบมีระเบียบ** — รูป 3:4 → จุดสีให้เลือก → ชื่อ → ราคา (ตัวหนา) → ดาว+จำนวนรีวิว → ไซส์ → สถานะสต๊อก เรียงตามนี้เสมอ
5. **Hero มีคน** — ภาพนางแบบ/นายแบบ + ป้ายคอลเลกชัน + ปุ่มชัดเจน ไม่ใช่ภาพผ้าหรือกราฟิกลอยๆ
6. **ทุก section มีหน้าที่ ไม่มี section ตกแต่งเปล่า** — ลำดับมาตรฐาน: announcement → header → hero → USP strip → หมวด → สินค้าแนะนำ → โปรโมชัน → เครื่องมือ (track order/วิธีจ่าย) → feature icons → footer เต็มรูปแบบ
7. **Rounded + เงาแบบจางมาก** — radius 8–12px, เงา `0 1px 3px rgba(0,0,0,.06)` เท่านั้น ไม่มีเงาฟุ้งลอย

## §1. กติการูปภาพ (ทำก่อนเขียนโค้ด — ข้อบังคับ)

**Shot list ขั้นต่ำต่อเดโม่ 1 ตัว** (โหลดจาก Unsplash/Pexels ซึ่ง license ใช้เชิงพาณิชย์ได้ เก็บลง `public/demo/`):
- Hero lifestyle มีคน 1–3 ภาพ (แนวนอนกว้าง ≥1800px)
- สินค้า on-model 8–12 ภาพ (แนวตั้ง 3:4)
- สินค้า flat-lay/แขวน 8–12 ภาพ (พื้นเรียบสีเดียว)
- ภาพหมวดหมู่ 6–10 ภาพ
- แบนเนอร์โปรโมชัน 2–3 ภาพ

**กติกาโทน (สำคัญกว่าจำนวน):** เลือกจากช่างภาพ/ชุดถ่ายเดียวกันให้มากที่สุด, โทนกลาง-อุ่น แสงธรรมชาติ, พื้นหลังเรียบ (ขาว/เทา/เบจ/ปูน), ห้ามปนภาพแสงสตูดิโอจัดจ้ากับแสงธรรมชาติ, ก่อนใช้ปรับให้เข้ากัน: ลด saturation เหลือ 88–92% ทุกภาพ
คีย์เวิร์ดค้นหา: `minimal fashion lookbook`, `beige outfit studio`, `clothing flat lay`, `linen shirt model`, `fashion ecommerce photography`

## §2. Design System ร่วม (ทุกเทมเพลตใช้ token ชุดเดียว ต่างที่ค่า)

```css
:root{
  --bg:#FFFFFF; --surface:#F7F7F7; --ink:#111214; --gray:#6B6F76; --line:#E8E8EA;
  --accent: /* ต่อเทมเพลต ดู §3 */;
  --danger:#D6453D;            /* ราคา sale / badge ลดราคา เท่านั้น */
  --radius-card:12px; --radius-btn:8px; --radius-pill:999px;
  --shadow-card:0 1px 3px rgba(0,0,0,.06);
  --font-head:'IBM Plex Sans Thai'; --font-body:'IBM Plex Sans Thai';  /* T4 ใช้ serif ดู §3.4 */
}
```
- ปุ่มหลัก: พื้น ink ตัวขาว สูง 48px / ปุ่มรอง: กรอบ 1px
- ดาวรีวิว: สีเดียว #F5A623 ขนาด 14px + "(จำนวน)" สี gray
- Badge: ใหม่=ink, ลดราคา=danger, BEST=#8B6F47 — pill 11px uppercase
- ระยะระหว่าง section: 72px (T1–T2), 64px (T3 เพราะแน่นกว่า), 96px (T4)
- Typography: หัว section 28–32px/600, ชื่อสินค้า 15px/500, ราคา 16px/700

## §3. สเปกรายเทมเพลต (4 ระดับ = 4 แพลนขาย)

### 3.1 T1 "SIMPLE" — แพลนเริ่มต้น ฿990 (ตาม ref: SIMPLE WEAR)
- accent: `#111214` (ไม่มีสีสัน — เรียบสุด), ตัวหนังสือโลโก้ + tagline ตัวเล็ก
- โครง: announcement bar ("ส่งสินค้าทั่วประเทศ | จัดส่งไว 1–3 วัน") → header: โลโก้ / เมนู 4 / **ปุ่ม LINE เขียว + ปุ่ม Facebook น้ำเงิน** ตัวเต็มมีไอคอน → hero split: ข้อความซ้าย (หัว 40px + sub + ปุ่มดำ) รูปนายแบบขวา พื้นเบจอ่อน → "สินค้าแนะนำ" 4 การ์ด (รูป, ชื่อ, ราคา, ปุ่มกรอบ "ดูรายละเอียด") → ปุ่มดำ "ดูสินค้าทั้งหมด" → แถบ CTA คู่ LINE/Facebook พื้นเขียวอ่อน → แถบ "ฟีเจอร์ของเว็บ" 4 ไอคอนพร้อมเลขข้อ → footer 3 คอลัมน์
- การ์ด T1 ไม่มีดาว/จุดสี/ไซส์ — เรียบตามแพลน
- หมายเหตุจากแพลนจริง: ถ้าลูกค้าซื้อ P1 ที่รวมระบบสั่งซื้อ+PromptPay ให้เปลี่ยนปุ่ม "ดูรายละเอียด" เป็น "สั่งซื้อ" และตัดแถบ "ไม่มีระบบตะกร้า" ออก — ตัดสินใจตาม feature flag ไม่ใช่ตายตัว

### 3.2 T2 "STORE" — แพลนร้านค้า ฿3,900 (ตาม ref: WEAR.STORE)
- accent: `#111214` + จุดสีสินค้า, โลโก้แบบ "WEAR.STORE" ตัวพิมพ์ใหญ่มีจุด
- เพิ่มจาก T1: utility bar บนสุด (ส่งฟรีเมื่อครบ / เปลี่ยนคืน 14 วัน / ติดตามคำสั่งซื้อ) → header มี search box จริง + ไอคอน บัญชี/wishlist/ตะกร้า(ตัวเลข) → hero เต็มกว้างภาพคู่นายแบบ-นางแบบ + "NEW COLLECTION SUMMER 2026" → แถบ USP 4 ไอคอน → สินค้าแนะนำ: การ์ดเต็มสูตร (badge "ใหม่", หัวใจ wishlist มุมขวา, จุดสี, ชื่อ, ราคา, ดาว+รีวิว) → แบนเนอร์หมวด 3 ใบ (รูปคน + ชื่อหมวด + ปุ่มเล็ก) → แถวเครื่องมือ 3 กล่อง: ติดตามคำสั่งซื้อ (ช่องกรอกเลข) / สถานะออร์เดอร์ล่าสุด (timeline 4 จุด สีเขียวถึงขั้นปัจจุบัน + เลขพัสดุ) / วิธีชำระเงิน (โลโก้บัตร + พร้อมเพย์) → แถบ "ครบทุกฟังก์ชัน" 5 ไอคอน → footer + newsletter
- Quick-view panel (ตาม ref มุมขวา): คลิกการ์ดเปิด panel เลือกสี/ไซส์/จำนวน + ปุ่ม "เพิ่มลงตะกร้า/ซื้อเลย"

### 3.3 T3 "HUB" — แพลนธุรกิจ ฿7,900 (ตาม ref: FASHION HUB)
- accent: `#D6453D` ใช้กับ SALE เท่านั้น
- ความต่างหลัก = **ความหนาแน่นแบบ marketplace**: แถวหมวดเป็นวงกลมรูปสินค้า 9–10 วง เลื่อนได้ → layout 2 คอลัมน์: sidebar ฟิลเตอร์ซ้าย (หมวด checkbox, แบรนด์+ช่องค้น, slider ราคา + ปุ่มช่วงราคา, จุดสี, ไซส์, สต๊อก) + grid สินค้า 5 คอลัมน์ขวา → การ์ดมี badge มุมซ้าย (NEW/-20%/BEST/-15%), แถวไซส์ S M L XL เป็นชิปเล็ก, "สต๊อก: พร้อมส่ง / เหลือ 3 ชิ้น" (เหลือน้อย = สี danger) → แถบสมาชิก 3 ช่อง (ส่วนลด 10% / ส่งฟรี / ผ่อน 0%) → แถวสินค้าแนะนำเลื่อนแนวนอน → "บทความแฟชั่น / Lookbook" การ์ด 3 ใบมีวันที่ → แถบ "ระบบและบริการ" 8 ไอคอน → footer มีโลโก้ payment ครบ
- hero เป็น carousel มีลูกศร + จุดบอกตำแหน่ง, member bar ใต้ header (สมาชิก Silver / คูปอง 3 ใบ / คะแนน 1,250)

### 3.4 T4 "LUXÉ" — แพลนพรีเมียม ฿15,900 (ตาม ref: LUXÉ + BRAND.CO)
- **เปลี่ยนบุคลิก:** --font-head เป็น `'Noto Serif Thai'` + Latin `Cormorant Garamond` โลโก้ตัวโปร่ง letter-spacing กว้าง, จังหวะหายใจกว้างขึ้น (section 96px)
- hero เต็มจอ ภาพแฟชั่นโทนเข้ม + "Timeless Elegance" serif 64px สีขาว + ปุ่มขาว/ปุ่มลิงก์ → USP strip พื้นเทาอ่อน → "ใหม่ล่าสุด" การ์ดพรีเมียม (รูปใหญ่ 3:4, ชื่อ, ราคา, จุดสี — **ไม่มีดาว/badge** ของหรูไม่โชว์รีวิว) → split section คู่: Lookbook (ภาพเต็ม+ข้อความทับ) / Brand Story (พื้น ink ตัวขาว + ลิงก์ "อ่านเรื่องราวของเรา →") → แถบไฮไลต์ 4 ไอคอนเส้นบาง → แถว Size Guide / ส่วนลดลูกค้าใหม่ WELCOME10 / newsletter → แถบ payment + SSL + "ลูกค้ากว่า 50,000+ คน" → footer ดำ "ทำไมต้องเลือก" 7 ไอคอน
- ถ้าลูกค้าต้องการระดับ BRAND.CO (omni-channel, แดชบอร์ด, สมาชิก tier): เป็นงาน custom เพิ่มจาก T4 — ระบุในใบเสนอราคาแยก ไม่รวมในเทมเพลต

## §4. Motion (เบา — ความว้าวมาจากรูป ไม่ใช่ animation)
- hover การ์ด: รูปซูม scale(1.04) 400ms + เงาเพิ่มเป็น `0 4px 16px rgba(0,0,0,.08)`
- hover การ์ด T2–T4: สลับรูปที่สอง (on-model ↔ flat-lay)
- ปุ่ม: พื้นเข้มขึ้น 150ms / hero T3: carousel เลื่อนอัตโนมัติ 5s หยุดเมื่อ hover
- ห้าม: parallax, ตัวอักษรวิ่ง, fade ทีละ section ตอน scroll ทั้งหน้า

## §5. สิ่งต้องห้าม (ฉบับ Commerce Premium)
1. รูป placeholder/กล่องสี/ภาพที่ไม่ใช่ภาพถ่ายจริง — blocker อันดับหนึ่ง
2. ภาพคนละโทนแสงปนกันใน section เดียว
3. gradient ทุกชนิดยกเว้น scrim บนภาพ, สีม่วง/น้ำเงินสด (ยกเว้นโลโก้ Facebook/payment)
4. เงาเกินค่าที่กำหนด 2 ค่า, radius เกิน 12px (ยกเว้น pill)
5. emoji ใน UI, Lorem ipsum, ราคา "0" หรือ "XXX"
6. ดาวรีวิว/badge บน T4 (ผิดบุคลิก luxury)
7. ฟอนต์เกิน 2 ตระกูลต่อเทมเพลต

## §6. Definition of Done (ต่อเทมเพลต)
1. Loop screenshot-วิจารณ์ ≥ 3 รอบ ที่ 390/768/1440px แนบคำวิจารณ์เป็นข้อความทุกรอบ
2. วางเทียบข้าง reference ภาพต้นแบบ แล้วตอบ: "องค์ประกอบไหนของ ref ที่ยังไม่มี" — ต้องเหลือศูนย์หรือมีเหตุผลบันทึกไว้
3. รูปจริงครบตาม shot list §1, โทนผ่านกติกา, ไม่มี placeholder
4. การ์ดสินค้ามีครบตามลำดับ anatomy §0 ข้อ 4 (ตาม tier)
5. มือถือ: การ์ด 2 คอลัมน์, hero ไม่ตัดหัวคน (ตรวจ object-position), เมนูเป็น drawer
6. Lighthouse mobile ≥ 85, CLS < 0.1, รูปทุกใบมี width/height กัน layout shift
7. คำถามปิดงาน: "ถ้าเอาไปวางข้าง 5 ภาพ ref ลูกค้าแยกออกไหมว่าอันไหนของจริงอันไหนเดโม่?" — ถ้าแยกออก ระบุจุดที่ฟ้องแล้วแก้
